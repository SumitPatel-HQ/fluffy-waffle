# **Hackathon 2025 — Domain Rules**

## **1\. Agentic AI — Domain Rules**

### **Submission Format**

* Teams must submit a public document containing snapshots of their AI workflows and development process.  
* The DOC must be public at the time of submission.

### **Submission Requirements**

* Complete DOC.  
* A README.md with project description and setup instructions.  
* A demo link, hosted URL, or screen recording.

### **Project Expectations**

* Projects must demonstrate agentic AI workflows, automation systems, or multi-step AI reasoning processes.  
* Teams may use LLM APIs, agent frameworks, workflow automation tools, or AI orchestration systems.  
* The focus should be on functional AI systems rather than only UI prototypes.

### **Original Work Requirement**

* Existing workflows, automation pipelines, or previously built solutions are strictly prohibited.  
* All workflows and core logic must be designed and implemented during the hackathon event period.  
* Teams may use public libraries, frameworks, and APIs, but the core implementation must be original.

## **2\. UI / UX — Domain Rules**

### **Submission Format**

* Teams must submit a Figma file link.  
* The file must be shared with the permission: 'Anyone with the link can view.'

### **Submission Requirements**

* A minimum of 5 screens or frames.  
* A clear user flow or navigation diagram.  
* A working Figma prototype.  
* Designs must be original and created during the event.

### **Project Expectations**

* Solutions should demonstrate thoughtful user-centered design.  
* The focus should be on usability, interaction quality, and visual consistency.

### **Template Restriction**

* Use of pre-made Figma design templates is strictly prohibited.  
* Teams must create original UI layouts and flows during the event.  
* Icons, fonts, and illustration libraries may be used if integrated into an original design.

## **3\. Vibe Coding — Domain Rules**

### **Submission Format**

* Teams must submit a public GitHub repository link.  
* The repository must be accessible to judges.

### **Submission Requirements**

* Full source code.  
* A README file with project overview.  
* Setup instructions to run the project.  
* A live demo link or screen recording.

### **Project Expectations**

* Projects should demonstrate rapid product development with strong visual polish.  
* Solutions should include functional features and a compelling user experience.

### **Original Work Requirement**

* Existing workflows or previously built project structures are strictly prohibited.  
* All features and implementations must be created during the hackathon event period.  
* Public libraries and frameworks are allowed, but the core project must be original.
